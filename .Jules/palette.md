## 2024-04-15 - Map Grid Keyboard Focus Issue
**Learning:** The interactive state map grids (`StateMap.tsx` and `StateMapGrid.tsx`) were inaccessible to keyboard users because the interactive elements were `<button>`s but only had `onMouseEnter`/`onMouseLeave` handlers to show state details, and lacked `onFocus`/`onBlur` handlers. Screen readers could focus the buttons, but sighted keyboard users wouldn't see the detail tooltip popups. Furthermore, copy-to-clipboard functionality lacked a polite screen reader announcement.
**Action:** Always ensure that interactive elements relying on hover states (`onMouseEnter`/`onMouseLeave`) also have corresponding focus states (`onFocus`/`onBlur`) for keyboard accessibility. Ensure copy buttons utilize an `aria-live="polite"` region to announce success to screen readers.
